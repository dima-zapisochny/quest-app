# Настройка Supabase для Quest App

## Шаги настройки

### 1. Создайте проект в Supabase

1. Перейдите на [supabase.com](https://supabase.com)
2. Войдите или зарегистрируйтесь
3. Создайте новый проект
4. Дождитесь завершения инициализации проекта

### 2. Создайте таблицы в базе данных

1. В Supabase Dashboard перейдите в **SQL Editor**
2. Откройте файл `supabase-schema.sql` из корня проекта
3. Скопируйте весь SQL-код из файла
4. Вставьте его в SQL Editor в Supabase
5. Нажмите **Run** для выполнения

Это создаст следующие таблицы:
- `user_profiles` - профили пользователей
- `quests` - квесты
- `game_sessions` - игровые сессии
- `quest_progress` - прогресс прохождения квестов

### 3. Получите ключи API

1. В Supabase Dashboard перейдите в **Settings** → **API**
2. Найдите следующие значения:
   - **Project URL** (например: `https://xxxxx.supabase.co`)
   - **anon public** ключ (в разделе Project API keys)

### 4. Настройте переменные окружения

1. Создайте файл `.env` в корне проекта (на основе `.env.example`)
2. Добавьте следующие переменные:

```env
VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_ANON_KEY=your-anon-key-here
```

Замените `your-project.supabase.co` и `your-anon-key-here` на ваши реальные значения.

### 5. Перезапустите приложение

```bash
npm run dev
```

## Проверка работы

После настройки:

1. Откройте приложение в браузере
2. Откройте консоль разработчика (F12)
3. Проверьте, что нет ошибок подключения к Supabase
4. Создайте тестовый квест - он должен сохраниться в Supabase
5. Проверьте в Supabase Dashboard → Table Editor, что данные появились

## Важные замечания

- **Row Level Security (RLS)** включен для всех таблиц с политиками, разрешающими чтение и запись для всех пользователей. В продакшене рекомендуется настроить более строгие политики безопасности.
- Данные автоматически синхронизируются между вкладками через **Real-time подписки** Supabase
- При ошибках подключения к Supabase приложение автоматически использует localStorage как fallback
- Прогресс квестов хранится отдельно в таблице `quest_progress` и привязывается к пользователю

## Миграция существующих данных

Если у вас уже есть данные в localStorage:

1. Экспортируйте данные через функцию экспорта в админ-панели
2. После настройки Supabase импортируйте данные обратно
3. Данные автоматически сохранятся в Supabase
