# Миграция: Добавление user_id к квестам

Эта миграция добавляет привязку квестов к пользователям, чтобы каждый пользователь видел и мог управлять только своими квестами.

## Шаги миграции

1. **Выполните SQL миграцию в Supabase SQL Editor:**

   Откройте файл `migrate-add-user-id-to-quests.sql` и выполните его содержимое в Supabase SQL Editor.

   Эта миграция:
   - Добавит поле `user_id` в таблицу `quests`
   - Создаст индекс для быстрого поиска квестов по пользователю
   - Обновит RLS политики для квестов

2. **Для существующих квестов:**

   Если у вас уже есть квесты в базе данных без `user_id`, их нужно либо удалить, либо назначить им владельца. Вы можете выполнить следующую команду для удаления всех квестов без владельца:

   ```sql
   DELETE FROM quests WHERE user_id IS NULL;
   ```

   Или назначить их конкретному пользователю (замените 'USER_ID' на реальный ID пользователя):

   ```sql
   UPDATE quests SET user_id = 'USER_ID' WHERE user_id IS NULL;
   ```

## Изменения в коде

После выполнения миграции:

- Каждый пользователь будет видеть только свои квесты
- При создании нового квеста он автоматически будет привязан к текущему пользователю
- Пользователи не смогут видеть или редактировать квесты других пользователей
- Все операции с квестами (создание, обновление, удаление) теперь требуют авторизации пользователя

## Откат миграции (если нужно)

Если нужно откатить изменения:

```sql
-- Удаляем индекс
DROP INDEX IF EXISTS idx_quests_user_id;

-- Удаляем поле user_id
ALTER TABLE quests DROP COLUMN IF EXISTS user_id;

-- Восстанавливаем старые политики (выполните из supabase-schema.sql, но удалите user_id из CREATE TABLE)
```
