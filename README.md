# Quiz Quest - Викторина-квест с админкой

Полноценное веб-приложение в формате «квиз-квеста»: игроки выбирают квест, переключаются между раундами и отвечают на вопросы с таймером и медиа. Админка позволяет в несколько кликов наполнить игру квестами, раундами, категориями и вопросами.

## 🚀 Технологии

- **Vue 3** (Composition API, `<script setup>`)
- **TypeScript**
- **Pinia** для глобального состояния
- **Vue Router**
- **Tailwind CSS** для UI
- **Vite** как сборщик
- **Vitest** для unit-тестов
- **localStorage** для хранения контента и прогресса

## 🎮 Игровой режим и роли

1. При входе на `/` укажите имя, выберите анимированный аватар животного и решите, что делать дальше.
2. **Создать игру** — перейдите к выбору квеста, получите четырёхсимвольный код и ведите игру с планшета или ноутбука.
3. **Присоединиться к игре** — введите код. На телефоне откроется экран игрока с большим «баззером».
4. Ведущий переключается между раундами, открывает вопросы и управляет очередью игроков прямо в модалке (кнопки «Правильно» / «Неправильно», просмотр медиа, пауза таймера).
5. Игроки видят только название квеста, описание текущего раунда, содержимое вопроса и огромную кнопку. Первый, кто нажал, подсвечивается зелёным, остальные — красным и ждут своей очереди. При неверном ответе таймер возобновляется, и можно нажимать снова.
6. Модалка занимает примерно 70 % экрана и поддерживает показ картинок и аудио для вопроса и ответа. При завершении таймера или нажатии «Показать ответ» ведущий и игроки одновременно видят правильный ответ.

## ⚙️ Управление квестами

1. Перейдите на `/host/setup`.
2. Создайте квест (название + описание) через кнопку "Создать квест".
3. После создания квеста вы автоматически перейдёте на страницу редактирования, где можно:
   - Добавить до 10 раундов. У каждого раунда можно задать название.
   - В раунде создавать категории (до 8) и вопросы (до 10). Вопросы поддерживают загрузку изображений и аудио как для текста вопроса, так и для ответа.
4. Все изменения моментально сохраняются в Pinia + Supabase.
5. Для редактирования существующего квеста нажмите на кнопку редактирования (карандаш) на карточке квеста.

```
src/

    admin/
      AdminRoundForm.vue        # Редактор раунда (категории, описание)

      AdminQuestionRow.vue      # Редактор вопроса, стоимость, медиа
    common/
      AvatarPicker.vue          # Выбор аватара с анимацией
  data/
  store/
    quizStore.ts                # Контент квестов + localStorage
    gameSessionStore.ts         # Игровые сессии, профили игроков и синхронизация с localStorage
  views/
    LandingView.vue             # Стартовый экран с регистрацией и выбором роли
    QuestsList.vue              # Каталог квестов для свободной игры
    QuestView.vue               # Игровой экран (работает и для хоста, и для свободной игры)
    PlayerSessionView.vue       # Экран игрока с «баззером»
    HostSetupView.vue           # Настройка и запуск сессии ведущего (также управление квестами)
    AdminQuestView.vue          # Редактор конкретного квеста
  router/
    index.ts                    # Роутинг
  App.vue
  main.ts
```

## 🛣️ Маршруты

- `/` — стартовый экран регистрации
- `/quests` — список квестов для одиночной игры
- `/quest/:questId` и `/quest/:questId/round/:roundId` — свободная игра
- `/host/setup` — выбор квеста для ведущего и управление квестами
- `/host/session/:sessionId/quest/:questId/round/:roundId?` — игровое поле ведущего
- `/session/:sessionId/player` — экран игрока с кнопкой
- `/admin/quest/:questId` — редактор выбранного квеста

## 🎨 UX и визуал

- Игровой режим — тёмная палитра в стиле ТВ-шоу. Плитки адаптивно масштабируются по высоте, не требуя скролла.
- Модалка вопроса занимает около 70 % экрана, поддерживает медиа и таймер. Для ведущего отображается очередь игроков и кнопки «Правильно/Неправильно».
- Экран игрока оптимизирован под мобильные устройства: крупный текст вопроса и полноэкранная кнопка-баззер, подсветка статуса.
- Админка — светлая, карточная сетка с подчеркнутой иерархией квест → раунды → категории → вопросы.
- На всех страницах предусмотрены кнопки-стрелки с подписью для возврата.

## 🔧 Разработка и расширение

1. Добавьте компонент в нужную директорию (`components/quiz`, `components/admin` или `components/common`).
2. При необходимости расширьте один из магазинов (`store/quizStore.ts` для контента, `store/gameSessionStore.ts` для сессий).
3. Обновите роуты (`router/index.ts`).
4. Оформите UI с Tailwind.
5. Напишите Vitest-тесты для новой логики.

### Хранение данных

- Контент квестов сохраняется под ключом `quiz-app-data`.
- Профиль пользователя и игровые сессии синхронизируются через `quiz-app-user-profile` и `quiz-app-runtime`, что позволяет корректно обновлять вкладки благодаря `storage`-событиям браузера.
- При первом запуске подставляется демонстрационный набор (музыкальный и исторический квесты).
